# Private Media Vault — Project Brief

## Overview

A fully client-side, privacy-first media aggregation and organization app. Built for a single user who wants to collect, organize, and access video content from multiple sources in one unified, secure interface—without leaving a trace.

---

## Problem Statement

Managing bookmarks, playlists, and media across multiple platforms is fragmented, cluttered, and leaves browser history/cookies. There's no clean way to:

- Aggregate content from different sources into one private library
- Organize with custom tags, ratings, and notes
- Track watch history without external tracking
- Access everything behind a simple lock

---

## Target User

- 26-year-old male
- Tech-comfortable but not necessarily a developer
- Values privacy and discretion
- Wants a fast, minimal, aesthetically pleasing tool
- Uses primarily on desktop (responsive is bonus, not priority)

---

## Core Features

### 1. Authentication Gate
- PIN or password lock on app load
- Auto-lock after configurable idle time
- No account creation—local only

### 2. Media Library
- Add media via URL (paste and auto-fetch metadata if possible)
- Bulk import from JSON/text file
- Thumbnail generation/caching via Canvas
- Grid, compact grid, and list view options

### 3. Organization
- Custom tags (create, edit, delete, color-code)
- Star rating (1-5)
- Favorites
- Personal notes per item
- Sort by: date added, rating, title, last viewed

### 4. Search & Filter
- Full-text search (title, tags, notes)
- Filter by tag, rating, favorites, date range
- Instant filtering—no server round-trips

### 5. Playback
- Slide-out player panel
- Embed support where possible (iframe)
- Playlist/queue functionality
- Track watch history with timestamps

### 6. Data Management
- All data stored in IndexedDB (via idb-keyval or Dexie)
- Export data as encrypted JSON
- Import from backup
- "Nuke" button to wipe all data instantly

### 7. Canvas-Specific Features
- Custom thumbnail rendering on Canvas
- Hover-to-preview animation (if source allows)
- Visual data indicators (rating stars, tag colors)
- Optional: image filter effects on thumbnails

---

## Non-Features (Out of Scope)

- Backend/server
- User accounts or multi-user support
- Cloud sync
- Mobile-first design (desktop priority)
- Video downloading or hosting
- DRM circumvention

---

## Tech Stack

| Layer         | Choice                                      |
|---------------|---------------------------------------------|
| Framework     | React 18+ (Vite)                            |
| State         | Zustand (lightweight, simple persistence)   |
| Storage       | IndexedDB via idb-keyval or Dexie.js        |
| Styling       | CSS Modules or Vanilla CSS (global.css)     |
| Canvas        | Native HTML5 Canvas API                     |
| Build         | Vite                                        |
| Optional      | Electron wrapper for desktop app            |

---

## Design Direction

- **iOS 26 Glassmorphism**: Layered translucent surfaces, blur, subtle depth
- **Dark mode only** (user preference assumed)
- **Minimal chrome**: Content is king, UI disappears
- **Fluid animations**: Spring-based, responsive, not jarring
- **Private by design**: No external requests unless user-initiated

## Success Metrics

- **Performance**: <100ms interaction latency, smooth 60fps animations
- **Privacy**: Zero network requests after initial load (unless fetching media)
- **Usability**: New media added in <10 seconds
- **Reliability**: Data persists across sessions, export/import works flawlessly
